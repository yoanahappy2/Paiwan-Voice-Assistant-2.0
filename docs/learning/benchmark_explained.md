# Benchmark 評測體系 — 技術解釋

## 一、為什麼要做 Benchmark？

比賽不是比誰功能多，是比誰能**證明**自己的技術有效。

沒有數據的技術報告 = 空話。
有數據的技術報告 = 「我試了 A、B、C 三種方案，BLEU 從 0 提升到 90%」。

學長說的對：**先建 Benchmark，再做優化，每一步都能衡量增益。**

---

## 二、三個指標是什麼？

### BLEU（0-1 分，越高越好）
機器翻譯的「考試分數」。把模型翻譯的結果跟標準答案比較。

- 看翻譯結果和答案重疊了多少詞
- 考慮 1-gram、2-gram、3-gram、4-gram 的匹配
- 1.0 = 完美匹配，0.0 = 完全不對
- 是翻譯領域最常用的指標

**白話：BLEU 就是「翻譯跟正確答案有多像」。**

### WER（詞錯誤率，越低越好）
模型輸出跟正確答案差了多少個詞。

- 用編輯距離計算：要把模型輸出改成正確答案，需要幾步操作（替換/刪除/插入）
- 除以總詞數，得到百分比
- 0 = 零錯誤，1.0 = 每個詞都錯

**白話：WER 就是「模型犯了多少錯」。**

- 我們的 baseline WER=2.57，意思是平均每個正確的詞旁邊多了 2.5 個錯的詞
- RAG 後 WER=0.018，幾乎零錯誤

### 完全匹配率（0-100%）
最直觀的「答對率」。

- 模型翻譯跟標準答案一模一樣才算對
- 最嚴格的標準
- 我們的測試集 40 題，RAG 答對了 36 題（90%）

## 三、四組實驗代表的故事

### 實驗 1 & 2：零樣本 Baseline
**問題：** 如果什麼都不教模型，它懂多少排灣語？

**答案：** 完全不懂。

- GLM-4-Flash：40 題全部零分
- GLM-4-Plus：40 題只蒙對 1 題（tiyamadju = 他們）
- 兩個模型都是 BLEU=0

**意義：** 證明排灣語在大模型訓練數據裡幾乎不存在。這不是模型的問題，是數據的問題。

### 實驗 3：RAG 關鍵詞檢索
**問題：** 如果把相關的排灣語-中文對照句餵給模型呢？

**答案：** 從 0% 跳到 90%。

原理：
1. 用戶輸入「masalu」
2. 在 242 條語料中搜索包含「masalu」的句子
3. 找到「masalu = 謝謝」
4. 把這條對照表連同問題一起給 LLM
5. LLM 看到對照表，自然就能翻譯

**意義：** 不需要訓練模型，不需要改參數，只是「給模型看參考資料」。242 條就夠了。

### 實驗 4：RAG 向量檢索
**問題：** 如果用語意相似度來搜索呢？

**答案：** 85%，比關鍵詞略低。

原理：
1. 用智譜 embedding-3 把所有語料轉成 2048 維向量
2. 用戶輸入也轉成向量
3. 計算餘弦相似度，找最接近的 Top-5 條
4. 餵給 LLM

為什麼比關鍵詞低？
- 短詞（「雲」「貓」「雷」）精確匹配效果本來就最好
- 向量搜索可能找到語意相近但不同的詞（「雲」可能找到「天空」）

向量搜索的優勢在哪？
- **能處理知識庫裡沒有的新句子**
- 例如用戶說「na tarivak sun ti kama?」，知識庫裡沒有這句，但向量搜索能找到語意最接近的問候句

## 四、最佳策略：混合 RAG

飛書 Bot 運行時的翻譯邏輯：

```
用戶輸入
    │
    ▼
Step 1: 精確匹配（快、準）
    │ 命中 → 直接回覆（90% 的情況）
    │ 沒命中 ↓
    ▼
Step 2: 向量語意搜索（能處理變體/新句）
    │ 找到 → 組合回覆
    │ 沒找到 ↓
    ▼
Step 3: LLM 零樣本翻譯（兜底）
```

這就是混合 RAG 架構，比單一方法更強。

## 五、這些數據在復賽怎麼用？

### 技術文件裡
> 「主流大模型零樣本排灣語翻譯 BLEU=0，加入 242 條 RAG 語料後提升至 90%。
> 證明低資源語言不需要昂貴的模型訓練，RAG 即可解決問題。」

### 答辯時
評審問：「你的系統效果怎麼樣？」
你答：「主流大模型完全不懂排灣語，BLEU 歸零。我們用 RAG 方法，242 條語料就做到 90% 完全匹配。而且這是可量化的，不是主觀判斷。」

### 對未來訓練的指導
```
Baseline (0%) → RAG (90%) → 未來加 SFT → 未來加 CPT
                  ↑ 
               我們在這裡
```

每一種新技術加進來，都用同一個 Benchmark 測，BLEU 升了多少就是增益。

- 如果 SFT 結果只有 85% → 不如直接用 RAG，錢白花了
- 如果 SFT 結果到 95% → 值得投入

**每一分錢和每一小時的投入都有數字衡量值不值得。**
