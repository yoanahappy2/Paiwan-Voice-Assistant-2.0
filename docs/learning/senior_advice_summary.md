## 學長的路線（馬來語項目）

1. 先用馬來語的評測集測試現有大模型 → 了解 baseline
2. 找到成型的馬來語基座模型 → 不從零開始
3. 從缺語料開始 → 收集語料 → 數據篩選 → 持續性訓練（CPT）
4. 基於已有模型迭代

## 關鍵建議

### RAG/SFT 的順序
> 「最好是先做 SFT，最好能先構建一個相應的 benchmark，然後再一步步優化。這樣每個環節能有多少增益都可以衡量。」

**正確順序：Benchmark → SFT → RAG**

### 用 RAG 做翻譯是可行的
排灣語目前沒有任何翻譯網站或軟體，RAG 方案填補了這個空白。

## 術語解釋

### CPT（Continual Pre-Training，持續預訓練）
- 在已訓練好的大模型上，繼續用新語料做預訓練
- 比如用排灣語語料繼續訓練一個中文/英文基座模型
- 比從零訓練便宜很多
- 適合低資源語言

### SFT（Supervised Fine-Tuning，監督微調）
- 用「問題-答案」配對的數據去訓練模型
- 目前做小語種最常用的方法
- 需要標注數據，質量 > 數量

### RL / RLHF（強化學習 / 人類反饋強化學習）
- 用人類反饋來微調模型
- ChatGPT 就是用 RLHF 調出來的
- 小語種場景：請母語者評分，用評分結果做 RL

### Benchmark（評測基準 / 測試集）
- 標準化的測試題目
- 馬來語有成型的評測集，排灣語目前幾乎沒有
- 常見指標：BLEU（翻譯品質）、WER（語音辨識錯誤率）

### 全量監督模型
- 大規模 SFT，用大量標注數據做監督微調
- 相對於 few-shot（只給幾個例子）

## 對我們項目的策略調整

### 原順序
1. 設計 baseline 評測集
2. 測試主流模型
3. 調研語料
4. 評估 CPT
5. 擴充 RAG

### 調整後順序
1. **Benchmark 優先** — 量化 baseline ✅ 已完成
2. **SFT** — 收集標注數據，用 benchmark 衡量提升（遠期）
3. **RAG** — 在 SFT 之後再加（我們提前做了，效果很好）
4. **CPT / RLHF** — 遠期

### 核心原則
- 每一步都用 benchmark 衡量增益
- SFT 先於 RAG（但我們反過來做也有效，因為語料太少不適合 SFT）
- 語料質量 > 數量
