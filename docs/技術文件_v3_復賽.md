## 第一章：問題定義

### 1.1 全球語言瀕危現狀

根據 UNESCO（聯合國教科文組織）統計，全球約 7,000 種語言中，超過 **40%** 面臨瀕危。平均每兩週就有一種語言消失。每一種語言的消亡，意味著獨特的文化認知、生態知識和世界觀的永久喪失。

### 1.2 台灣原住民語言的特殊困境

台灣 16 族原住民語言**全部**被列入瀕危名單。以排灣族語為例：

- **使用者銳減**：年輕一代的母語使用率不到 20%
- **缺乏數位資源**：主流 AI 模型（GPT-4、GLM-4 等）對排灣語的零樣本翻譯 BLEU 分數為 **0.0**
- **語料極度稀缺**：公開可用的排灣語數位語料僅數百句
- **教學資源匱乏**：合格的族語教師嚴重不足，學習場景受限

### 1.3 為什麼現有 AI 方案不適用

| 傳統 AI 方法 | 問題 |
|-------------|------|
| 大模型直接翻譯 | 零樣本 BLEU = 0，完全不懂排灣語 |
| 監督式微調大模型 | 需要數萬筆標註數據，不存在 |
| 蒐集大規模語料 | 成本高、時間長、瀕危語言等不起 |
| 商用語言學習 App | 不支援瀕危語言，無法定製 |

**核心矛盾**：瀕危語言最需要 AI 幫助，但瀕危語言恰恰缺乏訓練 AI 所需的數據。

## 第二章：方法論 — 極限語料下的 AI 框架

### 2.1 框架概述

我們提出一套**可遷移的瀕危語言 AI 框架**，在極少語料（100-500 句）下即可構建可用的語音識別、翻譯和教學服務。框架由四個協作的 Agent 組成：

```
┌─────────────────────────────────────────────┐
│         瀕危語言 AI 保護框架                 │
│                                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ ASR Agent│  │Translation│  │ Teaching │   │
│  │ 語音識別 │  │  Agent    │  │  Agent   │   │
│  │          │  │  翻譯服務 │  │ 教學服務 │   │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘   │
│       │             │              │          │
│       └─────────────┼──────────────┘          │
│                     │                         │
│              ┌──────┴──────┐                  │
│              │ 知識層      │                  │
│              │ RAG + 音韻  │                  │
│              │ + 語料庫    │                  │
│              └─────────────┘                  │
└─────────────────────────────────────────────┘
```

### 2.2 低資源 ASR：Whisper + LoRA 微調方法論

**基座模型選擇**：OpenAI Whisper-tiny（39M 參數）

選擇理由：
- Whisper 已在 680,000 小時多語言語音上預訓練，具備跨語言泛化能力
- tiny 版本僅 39M 參數，LoRA 微調只需 **2.3MB** 可訓練參數
- 可在消費級 GPU 上 38 秒完成訓練

**LoRA 微調策略**：
- Rank = 32, Alpha = 64
- 目標模組：encoder 的 self-attention 層
- 訓練數據：120 筆排灣族長老真實錄音（3 位發音人）

**音韻規則引擎**（三層後處理校正）：

| 層級 | 策略 | 範例 |
|------|------|------|
| 第一層 | 特定詞替換 | qerepus → 雲 |
| 第二層 | 音位互換（m/v） | miming → vuming |
| 第三層 | 音韻規則替換 | tj → ch, dj → j |

**質變效果**：

| 階段 | 準確率 | 語料量 | 提升 |
|------|--------|--------|------|
| Whisper-tiny 零樣本 | ~30% | 0 | — |
| + LoRA 微調 | 81.05% | 120 筆 | +51 百分點 |
| + 音韻校正 | ~90% | 120 筆 + 規則 | **+60 百分點總計** |

**關鍵洞察**：120 筆語料 + LoRA + 規則引擎，即可將準確率從 30% 提升到 90%。**這不是漸進改善，是質變。**

### 2.3 RAG 混合檢索：關鍵詞 + 向量

**核心發現**：主流大模型零樣本完全不懂排灣語。

| 模型 | 方法 | BLEU | 完全匹配率 |
|------|------|------|-----------|
| GLM-4-Flash | 零樣本 | 0.000 | 0.0% |
| GLM-4-Plus | 零樣本 | 0.000 | 2.5% |

**RAG 解決方案**：用極少語料建立檢索增強生成系統。

混合檢索策略：
1. **關鍵詞檢索**：精確匹配排灣語詞彙，對短詞效果最好
2. **向量檢索**：智譜 embedding-3（2048 維）+ FAISS IndexFlatL2，處理語義相似度
3. **混合策略**：關鍵詞優先（權重 ×2），向量補充（權重 ×1），分數融合排序

**語料擴充路徑**：

| 版本 | 語料量 | 來源 |
|------|--------|------|
| v1 | 117 句 | intent_training.jsonl |
| v2 | 242 條 | v1 + paiwan_phrases + CSV 詞彙表 |
| merged | **4,844 條** | v2 + klokah 爬蟲（對話/文化/散文/歌謠等 13 類） |

### 2.4 可遷移性論證

本框架的每個組件都是**語言無關**的：

| 組件 | 可遷移性 | 遷移成本 |
|------|----------|----------|
| Whisper + LoRA | 任何有音標/文字系統的語言 | 收集 100-200 句錄音 |
| 音韻規則引擎 | 需要語言學知識編寫規則 | 語言學家合作 1-2 天 |
| RAG 混合檢索 | 任何有雙語對照語料的語言 | 整理 200-500 句對照 |
| LLM 角色對話 | 更換 system prompt | 10 分鐘 |
| TTS（XTTS v2） | 任何有參考音頻的語言 | 10-50 句參考錄音 |

**遷移時間估算**：對於一個新的瀕危語言，若已有基礎語料（200 句錄音 + 500 句雙語對照），可在 **1-2 週**內建立完整的 AI 服務。

## 第三章：實驗與數據

### 3.1 翻譯品質對比

**測試集**：40 組排灣語-中文對照，覆蓋 7 個類別（詞彙/對話句/問候/文化/詩歌等）

| # | 實驗 | 語料量 | BLEU | WER | 完全匹配率 |
|---|------|--------|------|-----|-----------|
| 1 | GLM-4-Flash 零樣本 | 0 | 0.000 | 2.57 | 0.0% |
| 2 | GLM-4-Plus 零樣本 | 0 | 0.000 | 1.08 | 2.5% |
| 3 | + RAG 關鍵詞 (v2) | 242 | 0.452 | 0.018 | 90.0% |
| 4 | + RAG 向量 (v2) | 242 | 0.416 | 0.035 | 85.0% |
| 5 | + RAG 關鍵詞 (merged) | 4,844 | **0.916** | — | 82.5% |
| 6 | + RAG 向量 (merged) | 4,844 | 0.852 | — | 72.5% |
| 7 | + RAG 混合 (merged) | 4,844 | 0.873 | — | 77.5% |

**關鍵結論**：
1. **242 條語料讓 BLEU 從 0 跳到 0.45**（完全匹配率 0% → 90%）
2. **語料擴充到 4,844 條後 BLEU 達到 0.916**（翻譯品質翻倍提升）
3. **關鍵詞檢索在所有語料規模下都優於向量檢索**（精確匹配對低資源語言更有效）
4. **不需要訓練大模型**，RAG 是低資源語言翻譯的性價比最高方案

### 3.2 ASR 準確率

| 階段 | 準確率 | 語料量 |
|------|--------|--------|
| Whisper-tiny 零樣本 | ~30% | 0 |
| + LoRA 微調（120 筆） | 81.05% | 120 |
| + 三層音韻校正 | ~90% | 120 + 規則 |
| 噪音容忍度（輕微） | 96.2% | — |
| 噪音容忍度（中度） | 94.5% | — |

### 3.3 其他模組指標

| 模組 | 指標 | 值 |
|------|------|-----|
| 意圖分類 | 準確率（12 類） | 95.2% |
| 綴詞分析 | 檢測率（11 種前綴） | 52.4% |
| RAG 檢索 | Top-3 準確率 | 90.0% |
| 端到端延遲 | ASR → RAG → LLM | 3-4 秒 |

## 第四章：系統實現

### 4.1 架構圖

```
飛書用戶端（Bot / 文檔 / 多維表格）
    │
    ▼ WebSocket 長連接 / HTTP API
語聲同行 Bot 後端（Python + lark-oapi SDK）
    │
    ├→ ASR Agent：Whisper-tiny + LoRA → 音韻校正 → 文字
    ├→ Translation Agent：RAG 混合檢索 → LLM → 雙向翻譯
    ├→ Teaching Agent：語料庫 + 意圖分類 → 教學/測驗/每日一句
    └→ 知識層：4,844 條語料 + FAISS 索引 + 音韻規則
```

### 4.2 飛書原生整合

- **飛書 Bot**：WebSocket 長連接（無需公網 IP），支持文字 + 語音
- **指令系統**：`/help`、`/translate`、`/learn`、`/quiz`、`/daily`
- **vuvu Maliq 角色**：排灣族祖母語伴，混合排灣語 + 中文回覆
- **多維表格整合**：語音轉寫結果同步至飛書多維表格（開發中）

### 4.3 語料來源與規模

| 來源 | 數量 | 類型 |
|------|------|------|
| 排灣族長老錄音 | 120 筆 | WAV 音頻 |
| paiwan_phrases.txt | 118 句 | 日常對話 |
| paiwan_corpus.json | 42 句 | 結構化語料（7 類） |
| CSV 詞彙表 | 117 詞 | 數字/身體/親屬/動物/自然 |
| klokah 族語學習平台 | ~4,500 條 | 對話/文化/散文/歌謠等 13 類 |
| **合計（去重後）** | **4,844 條** | |

## 第五章：社會影響與未來工作

### 5.1 社會價值

本框架證明：**瀕危語言不需要等待大規模語料收集，用 200 句話就能建立可用的 AI 服務。** 這對全球 3,000+ 種瀕危語言的保存工作具有實際意義。

### 5.2 可持續性

- 用戶每次使用都在貢獻語料（語音轉寫→多維表格→語料庫）
- 形成正向循環：使用越多 → 語料越多 → 服務越好

### 5.3 擴展計畫

| 階段 | 目標 | 時間 |
|------|------|------|
| 短期 | 飛書 Bot 穩定運行 + 用戶測試 | 2026 Q2 |
| 中期 | 擴展至阿美族語、泰雅族語 | 2026 Q3-Q4 |
| 長期 | 覆蓋台灣 16 族原住民語言 | 2027 |
| 願景 | 開源框架，供全球瀕危語言社區使用 | — |
