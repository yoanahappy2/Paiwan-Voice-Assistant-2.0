# 基於 Agent-based 模組化架構的瀕危語言 AI 保護框架

## 復賽技術文件 v4

### 術語定義

本文件中「Agent」指**任務導向的專家模組**——每個 Agent 負責一個獨立功能域（語音辨識、翻譯、教學、口語練習、語料增長），由中央路由統一調度。不同於通用 Multi-Agent 框架，我們的 Agent 核心優勢在於將**語言學規則**（音韻引擎、綴詞分析）注入 AI 流程中，以解決低資源語言的幻覺問題。所有 Agent 共享同一套 RAG 知識庫與飛書 Bitable 數據中控台，形成「用戶使用 → 數據回流 → 模型優化」的完整閉環。

## 第一章：問題定義

### 1.1 全球語言瀕危現狀

根據 UNESCO（聯合國教科文組織）統計，全球約 7,000 種語言中，超過 **40%** 面臨瀕危。平均每兩週就有一種語言消失。每一種語言的消亡，意味著獨特的文化認知、生態知識和世界觀的永久喪失。

### 1.2 台灣原住民語言的特殊困境

台灣 16 族原住民語言**全部**被列入瀕危名單。以排灣族語為例：

- **使用者銳減**：年輕一代的母語使用率不到 20%
- **缺乏數位資源**：主流 AI 模型（GPT-4、GLM-4 等）對排灣語的零樣本翻譯 BLEU 分數為 **0.0**
- **語料極度稀缺**：公開可用的排灣語數位語料僅數百句
- **教學資源匱乏**：合格的族語教師嚴重不足，學習場景受限

### 1.3 為什麼現有 AI 方案不適用

- **大模型直接翻譯**：零樣本 BLEU = 0，完全不懂排灣語
- **In-Context Learning（ICL）**：5-shot、10-shot、20-shot BLEU 全部為 0（實驗驗證）
- **監督式微調大模型**：需要數萬筆標註數據，不存在
- **蒐集大規模語料**：成本高、時間長、瀕危語言等不起
- **商用語言學習 App**：不支援瀕危語言，無法定製

**核心矛盾**：瀕危語言最需要 AI 幫助，但瀕危語言恰恰缺乏訓練 AI 所需的數據。

### 2.2 低資源 ASR：Whisper-small + LoRA 微調

**基座模型升級**：從 Whisper-tiny（39M 參數）升級至 **Whisper-small**（244M 參數）

升級理由：
- Whisper 已在 680,000 小時多語言語音上預訓練，small 的跨語言泛化能力遠優於 tiny
- small 版本的 encoder-decoder 結構更完整，能捕捉排灣語獨特的音素特徵
- LoRA adapter 僅需 **14MB**（vs tiny 的 2.3MB），仍可在消費級 GPU 上訓練

**LoRA 微調策略**：
- Rank = 32, Alpha = 64
- 目標模組：encoder 的 self-attention 層
- 訓練數據：120 筆排灣族長老真實錄音（3 位發音人）
- **關鍵技術突破**：解決 peft + Whisper 的相容性問題——`PeftModel.forward()` 會把 `labels` 映射成 `input_ids`，解法是不設定 `task_type` 並使用原生 PyTorch 訓練迴圈

**音韻規則引擎**（三層後處理校正）：

- **第一層**：特定詞替換（如排灣語特有詞彙的對應修正）
- **第二層**：音位互換（如 m/v 互換，排灣語方言變體）
- **第三層**：音韻規則替換（tj→ch, dj→j, lj→ly, q→k, c→ts）

**質變效果**：

| 階段 | 準確率 | 語料量 | 提升 |
|------|--------|--------|------|
| Whisper-tiny 零樣本 | ~30% | 0 | — |
| Whisper-tiny + LoRA | 81% | 120 筆 | +51 百分點 |
| Whisper-small + LoRA | **96.8%** | 120 筆 | **+67 百分點** |
| 可靠辨識詞數 | **94 個高準確詞** | — | — |

**關鍵洞察**：同樣 120 筆語料，僅升級基座模型就從 81% 跳到 96.8%。模型選擇比數據量更重要。

### 2.3 RAG 混合檢索：關鍵詞 + 向量

**核心發現**：主流大模型零樣本完全不懂排灣語，ICL 也無效。

| 模型 | 方法 | BLEU | 完全匹配率 |
|------|------|------|-----------|
| GLM-4-Flash | 零樣本 | 0.000 | 0.0% |
| GLM-4-Plus | 零樣本 | 0.000 | 2.5% |
| GLM-4-Flash | ICL 5-shot | 0.000 | 0.0% |
| GLM-4-Flash | ICL 10-shot | 0.000 | 0.0% |
| GLM-4-Flash | ICL 20-shot | 0.000 | 0.0% |

> ICL（In-Context Learning）實驗證實：即使提供 20 個排灣語範例作為 context，模型仍然完全無法翻譯。這與 Li & Niehues (2025) 的研究結論一致——低資源語言需要檢索增強而非上下文學習。

**RAG 解決方案**：用極少語料建立檢索增強生成系統。

混合檢索策略：
1. **關鍵詞檢索**：精確匹配排灣語詞彙，對短詞效果最好
2. **向量檢索**：智譜 embedding-3（2048 維）+ FAISS IndexFlatL2，處理語義相似度
3. **混合策略**：關鍵詞優先（權重 ×2），向量補充（權重 ×1），分數融合排序

**語料擴充路徑**：

| 版本 | 語料量 | 來源 |
|------|--------|------|
| v1 | 117 句 | intent_training.jsonl |
| v2 | 242 條 | v1 + paiwan_phrases + CSV 詞彙表 |
| merged | **4,844 條** | v2 + klokah 爬蟲（對話/文化/散文/歌謠等 13 類） |

### 2.4 分層語音策略

排灣語不在任何主流 TTS 系統的支援語言中。我們提出三層語音方案：

| 層級 | 來源 | 數量 | 品質 | 用途 |
|------|------|------|------|------|
| Layer 1 | 真人錄音（s03 母語者） | 101 個 | 最高 | 優先使用 |
| Layer 2 | XTTS v2（ASR 驗證通過） | 28 個 | 中等 | 擴充覆蓋 |
| Layer 3 | XTTS v2（未通過驗證） | 22 個 | 備用 | 降級使用 |

**TTS 品質驗證閉環**：用自訓練 ASR 對 TTS 輸出做反向驗證（TTS 生成 → ASR 辨識 → 比對原文），量化評估 TTS 品質：
- V1（原文輸入）：26/50 完全正確（52%），平均準確率 68.7%
- V2（IPA 轉換）：額外修好 2 詞（IPA 轉換規則：tj→ch, q→k, c→ts, lj→ly, dj→j）

**學術價值**：分層語音策略 + ASR 反向驗證閉環，是低資源瀕危語言 TTS 的可行工程方案。

### 2.5 知識圖譜引擎

建立排灣語詞彙的語義關聯網絡，支援智能推薦和查詢：

- **10 個詞綴前綴規則**：識別排灣語的構詞法（如 -an 表處所、si- 表製造）
- **13 個親屬稱謂關聯**：排灣語親屬系統的結構化表示
- **7 個語義群組**：數字、身體、自然、動物、親屬、問候、日常
- **47 條詞彙雙向關聯**：查一個詞自動推薦同群組相關詞

### 2.6 可遷移性論證

本框架的每個組件都是**語言無關**的：

| 組件 | 可遷移性 | 遷移成本 |
|------|----------|----------|
| Whisper + LoRA | 任何有音標/文字系統的語言 | 收集 100-200 句錄音 |
| 音韻規則引擎 | 需要語言學知識編寫規則 | 語言學家合作 1-2 天 |
| RAG 混合檢索 | 任何有雙語對照語料的語言 | 整理 200-500 句對照 |
| 知識圖譜 | 任何有語義群組的語言 | 整理詞彙關聯 1-2 天 |
| 分層語音策略 | 任何有參考音頻的語言 | 10-50 句參考錄音 |
| 主動學習閉環 | 通用 | 即插即用 |
| Bitable 儀表板 | 通用 | 配置表格結構 |

**遷移時間估算**：對於一個新的瀕危語言，若已有基礎語料（200 句錄音 + 500 句雙語對照），可在 **1-2 週**內建立完整的 AI 服務。

## 第四章：系統實現

### 4.1 架構圖

```
飛書用戶端（Bot 語音/文字 + Bitable 多維表格）
    │
    ▼ WebSocket 長連接（無需公網 IP）
語聲同行 Bot 後端（Python + lark-oapi SDK）
    │
    ├→ ASR Agent：Whisper-small + LoRA → 音韻校正 → 文字（96.8%）
    ├→ Translation Agent：RAG 混合檢索 → LLM → 雙向翻譯（BLEU 0.916）
    ├→ Teaching Agent：語料庫 + 意圖分類 → 教學/測驗/每日一句
    ├→ Speaking Agent：語音播放 → 用戶跟讀 → ASR 辨識 → 模糊匹配 → 評價
    ├→ Growth Agent：主動學習 + 社群語料 + 語料回流
    └→ 知識層：4,844 條語料 + FAISS 索引 + 音韻規則 + 知識圖譜
         │
         ▼ Bitable 儀表板
    學習熱力圖 + 社群語料審核 + 詞彙掌握統計
```

### 4.2 飛書 Bot 指令系統（9 個指令）

| 指令 | 功能 | AI 技術 |
|------|------|---------|
| `/help` | 顯示所有指令列表 | — |
| `/translate` | 中↔排灣雙向翻譯 + 語音 | RAG 混合檢索 + LLM |
| `/learn` | 智能推薦新詞 + 語音卡片 | UserProfile + 知識圖譜推薦 |
| `/quiz` | 出題測驗 + 自動記錄 | 意圖分類 + Bitable 寫入 |
| `/daily` | 每日一句 + 語音 | RAG + 分層語音 |
| `/speak` | 口語練習（聽→說→辨識→評價） | ASR + 模糊匹配 |
| `/lookup` | 知識圖譜查詢 + 相關詞推薦 | 知識圖譜引擎 |
| `/submit` | 提交新詞到社群語料 | Bitable 寫入 + 去重 |
| `/stats` | 學習報告 + 7天熱力圖 | Bitable 讀取 + 統計 |

**vuvu Maliq 角色**：排灣族祖母語伴，混合排灣語 + 中文回覆，增加文化沉浸感。

### 4.3 口語練習閉環

完整的「聽→說→辨識→評價」語音學習閉環：

1. **/speak** 指令：Bot 發送語音（真人錄音優先）+ 文字提示
2. **用戶跟讀**：長按語音按鈕念出排灣語
3. **ASR 辨識**：自訓練 Whisper-small + LoRA 辨識用戶發音
4. **模糊匹配**：difflib 比對辨識結果與目標詞彙
5. **評價回饋**：給出正確/接近/再試一次的評價

**技術亮點**：ASR 不是調第三方 API，是自訓練模型。辨識→評價全在本地完成，延遲低於 2 秒。

### 4.4 Bitable 儀表板（3 張表）

多維表格不只是記錄器，是活的社群語言保存平台：

**表 1：學習熱力圖**（`tblcuoXMUJzGWW53`）
- 每個用戶每天的學習軌跡
- 欄位：日期、學習詞數、翻譯次數、測驗結果、活躍度等級
- 活躍度分級：🔥 高度活躍（≥15）/ ☀️ 活躍（≥8）/ ☁️ 普通（≥3）/ ❄️ 低活躍（<3）

**表 2：社群語料審核**（`tbl3poVuE8oRwHBv`）
- 用戶提交新詞 → 母語者審核 → 回流語料庫
- 支持去重（已有詞彙不自動重複提交）
- 實現「用戶貢獻 → 社群審核 → 系統增長」的良性循環

**表 3：詞彙掌握統計**（`tblvggxM4d0kx1AK`）
- 每個詞的學習人數、正確率、掌握度
- 掌握度分級：🟢 已掌握（≥80%）/ 🟡 學習中（≥50%）/ 🔴 困難詞（<50%）/ ⚪ 新詞

### 4.5 主動學習閉環

實現語料自動增長引擎：

- **`active_learner.py`**：監控用戶翻譯對話，高置信度翻譯自動標記為候選語料
- **`retrain_pipeline.py`**：用戶確認後，候選語料回流至 RAG 語料庫，觸發增量重訓練
- **流程**：用戶翻譯 → RAG 檢索 → LLM 生成 → 置信度評估 → 用戶確認 → 語料回流

## 第五章：社會影響與未來工作

### 5.1 社會價值

本框架證明：**瀕危語言不需要等待大規模語料收集，用 200 句話就能建立可用的 AI 服務。** 這對全球 3,000+ 種瀕危語言的保存工作具有實際意義。

### 5.2 可持續性：社群語料農場

- 用戶每次使用都在貢獻語料（學習/翻譯/測驗 → Bitable → 語料庫）
- 母語者可通過 Bitable 審核社群提交的詞彙
- 主動學習閉環自動發現高質量候選語料
- 形成正向循環：**使用越多 → 語料越多 → 服務越好**

### 5.3 擴展計畫

| 階段 | 目標 | 時間 |
|------|------|------|
| 短期 | 飛書 Bot 穩定運行 + 擴大用戶測試 | 2026 Q2 |
| 中期 | 擴展至阿美族語、泰雅族語 | 2026 Q3-Q4 |
| 長期 | 覆蓋台灣 16 族原住民語言 | 2027 |
| 願景 | 開源框架，供全球瀕危語言社區使用 | — |
